0What is Phase A?

Phase A is the hypothesis formation step of the clean-room ensemble launch plan (ensemble-launch-clean-room-2026-07.md). It answers: which open-source model panels should we try? — not which panel wins?

Clean room means every model name, price, and score was pulled fresh from public APIs and leaderboards on 2026-07-07. Selection rules were written before looking at rankings. Prior internal shortlists and experiment conclusions were deliberately not used as input. If Phase A lands on similar panels to older docs, that is convergence; if not, we learned the old lists were stale.

Phase A produces three committed artifact types:

  • Catalog snapshot — every candidate OSS coding model on OpenRouter with slug, price, context, provider options, lineage tag, and third-party benchmark scores (each with URL).
  • Shortlist (8–12 models) — ranked by a pre-committed simple mean of public scores. Used only for shortlisting, not for launch claims.
  • Hypothesis cards (H1–H5) — falsifiable panel configs with pinned model IDs, judge, cost projections, predictions, and kill conditions. Written before any billed run.

Phase A spends $0 on API calls. The first real measurements happen in Phase C (~$25–75) on a fixed task manifest we grade ourselves. Nothing in this briefing is publishable as a product score.

Cost formula (per model, per request):
cost = (2,000 × input_price_per_M + 8,000 × output_price_per_M) / 1,000,000
Prices come from the OpenRouter API fields pricing.prompt and pricing.completion, converted to USD per million tokens in the snapshot YAML. This is a planning estimate for comparing models — actual Phase C spend depends on real output lengths per task.

3Evidence coverage: how much do we actually know?

Step 3 of Phase A: collect third-party benchmark aggregates for every candidate. Each score in the table links to the exact leaderboard page it came from. Hover a cell for the full harness description and retrieval date.

Most important caveat: 25 of 34 candidates have at least one third-party score; 3 have none at all (nvidia/llama-3.3-nemotron-super-49b-v1.5, z-ai/glm-5-turbo, z-ai/glm-5v-turbo). 6How to read all of this

  • Nothing here is a measured result. Phase A produces hypotheses, not pass rates on our product harness. The shortlist mean is a shortlisting heuristic over incompatible public harnesses. Treat it as "which models are worth spending Phase C money on," not "which model is best."
  • What each hypothesis tests:
    • H1 — Does the honest top-3 backbone beat best-single?
    • H2 — Does near-tie style diversity beat pure rank?
    • H3 — Does cheap-first cascade win on $/solve? (deferred until cascade exists)
    • H4 — Does one strong model × 3 samples make panels pointless? Mandatory baseline.
    • H5 — Do 64k thinking budgets buy measurable headroom?
  • Acceptable outcomes: "H4 wins → ship routing preset, not fusion" is a fully valid shippable verdict. "All panels lose → back to drawing board" is also valid. The kill conditions on each card define what falsifies each bet.
  • What comes next (Phase B): emit FusionKit fusion configs from these cards, run cent-scale smoke tests (model IDs resolve, streaming works), and commit a measurement preregistration before Phase C spends real money.
  • Regenerate this page after any snapshot or card change: uv run python analysis/phase-a-briefing-2026-07-07/scripts/build_briefing.py

8Glossary

Panel
2–4 models that all attempt the same coding task in parallel before a judge merges answers.
Ensemble / fusion
A panel plus the machinery that merges answers (topology + judge). What FusionKit sells as a named product.
Judge / synthesizer
The model that reads all panel members' answers and produces one final response.
Self-MoA (H4)
One strong model sampled K times; keep the best answer via execution-guided grading. No judge.
Lineage veto
At most one model per base family per panel — clones from the same teacher fail on the same tasks.
Aggregate mean
Simple unweighted mean of third-party, unsaturated benchmark scores for shortlisting only.
Oracle (panel ceiling)
Theoretical max: pass if any panel member passes. Diagnostic only — computed in Phase C, not Phase A.
Headroom
Oracle minus best-single pass rate. Room for fusion to help if members fail on different tasks.
Truncation refusal
If >10% of a model's answers are cut off mid-stream at the practical budget, its score is refused (Phase C rule).
Evidence card
Publishable one-pager with measured score + cost — only Phase D produces these.
