---
title: Troubleshooting
description: Common failures and how fusionkit reports them.
---

Start with:

```bash
fusionkit doctor
```

It checks `uv`/`uvx`, the agent binaries, provider keys, git, and your repo
config, with a fix hint for anything missing.

## "preflight failed"

A required binary or key is missing. The message lists each problem with an
install hint, e.g. missing `uvx`, a missing agent (`codex`/`claude`/`cursor-agent`),
or an unset provider key. Install the binary or export the key and retry.

## "no --repo given and the current directory is not a git repository"

Run fusionkit from inside your project's git repo, or pass `--repo <dir>`.

## A panel model won't start

If a provider rejects the request (bad key or an inaccessible model), fusionkit
fails fast with a clear message naming the model and provider, instead of
retrying forever or dumping a stack trace:

```
panel model "gpt" (openai:gpt-5.5) could not start.
  Check the model name and that OPENAI_API_KEY grants access.
  ...
```

Transient startup failures are retried automatically with backoff.

## `--observe` says the dashboard isn't available

The dashboard ships separately from the npm package. Follow the printed
instructions to install its dependencies once. See
[Observability](/docs/guides/observability).

## Cursor needs a logged-in CLI

The `cursor` tool spawns the bundled Cursorkit bridge, so no separate checkout is
required. It only needs a logged-in `cursor-agent` CLI
([install](https://cursor.com/cli)) on your PATH. See [Cursor](/docs/tools/cursor).

## Cursor says "Access to private networks is forbidden"

Cursor's BYOK ("Override OpenAI Base URL") requests are proxied through
Cursor's backend, which deliberately blocks loopback and private addresses
(`ssrf_blocked`). A `http://127.0.0.1:...` base URL can never work. Use
`fusionkit serve --expose` to publish the gateway on a public HTTPS tunnel
with a required bearer token; it prints the exact Cursor settings. You can
also bring your own tunnel and pass `--public-url` or set
`FUSIONKIT_PUBLIC_URL`. For Cursor without BYOK, `fusionkit cursor --ide`
needs no tunnel at all. See [Cursor](/docs/tools/cursor).

## Where are the logs?

Each run writes full per-process logs to a temp directory; fusionkit prints the
path (`fusion: logs in ...`). Failures also surface the most relevant lines
inline.

## Keys are in `.env` but not picked up

fusionkit auto-loads `.env` from the current directory and the repo root, but
**already-exported** variables always win. If you exported an empty value, that
empty value is used. Unset it so the `.env` value applies.
