## Why cost matters here

The default panel runs **multiple frontier cloud models plus a judge** on every
prompt. That is the point — fusion trades extra calls for better answers — but it
means usage adds up faster than a single-model session.

fusionkit asks for a one-line confirmation before starting a cloud panel on an
interactive terminal. Skip it with `--yes` (or in non-interactive contexts).

## Ways to control cost

### Use the local panel

On Apple Silicon, run entirely on-device with no API spend:

```bash
fusionkit codex --local
```

### Pick cheaper models

Define your own panel:

```bash
fusionkit codex \
  --model fast=openai:gpt-5.5-mini \
  --model alt=anthropic:claude-haiku-4 \
  --judge-model openai:gpt-5.5-mini
```

Or commit it via [`fusionkit.json`](/docs/configuration).

### Shrink the panel

A panel can be a single model plus a judge if you want fusion's synthesis step
without a wide fan-out.

## Preview before you spend

```bash
fusionkit status
```

shows the effective panel, judge, and a note when a run will use cloud models.

<Callout type="warn">
  Model IDs in the defaults (e.g. `gpt-5.5`, `claude-sonnet-4-6`) must be enabled
  on your account. If a model is not accessible, fusionkit reports which model and
  provider failed rather than a raw stack trace.
</Callout>
