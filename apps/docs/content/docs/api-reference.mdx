The model-fusion service surface is defined by an **OpenAPI 3.1 contract** that
lives with the protocol package — it is the single source of truth, so this
reference is generated from it rather than written by hand.

## Generate the reference

The reference pages are produced from
`packages/protocol/openapi/model-fusion-harness-executor.openapi.json` using
[`fumadocs-openapi`](https://fumadocs.dev/docs/ui/openapi):

```bash
pnpm generate:openapi
```

This emits MDX into `content/docs/api`, which appears in the sidebar after a
rebuild. Regenerate whenever the contract changes so the docs never drift from
the spec.

## Why generated?

- The contract is versioned and validated in CI; copying it into prose would
  immediately drift.
- `fumadocs-openapi` renders request/response schemas, examples, and try-it
  panels directly from the spec.

<Callout type="info">
  The durable record shapes (rows, receipts, candidates, judge syntheses) are
  defined as JSON Schema in the same protocol package; the OpenAPI document
  references those records rather than re-declaring them.
</Callout>
