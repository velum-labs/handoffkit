## The `.fusionkit/` folder

Fusion settings live in a committed `.fusionkit/` folder at your repo root. It is
**safe to commit**. It stores only the env-var *names* that hold API keys
(`keyEnv`), never the secret values.

```
.fusionkit/
  fusion.json        # all settings (panel, judge, default tool, run defaults)
  prompts/
    judge.md                    # judge analysis system prompt
    synthesizer.md              # synthesizer system prompt (the fused output)
```

Scaffold it interactively:

```bash
fusionkit init
```

`init` writes `.fusionkit/fusion.json` and ejects the synthesizer's built-in
prompts into `.fusionkit/prompts/*.md` so you can edit them. Inspect the
effective configuration and a dry-run preview any time:

```bash
fusionkit status
```

<Callout type="info">
  A legacy `fusionkit.json` at the repo root is **auto-migrated** into
  `.fusionkit/fusion.json` on first run (the original is left in place as a
  fallback). You can delete it once the folder is committed.
</Callout>

## Precedence

Settings resolve in this order (highest wins):

```
explicit CLI flags  >  .fusionkit  >  built-in defaults
```

So the folder is a default layer, not a lock. Any flag overrides it for a
single run. Use `--no-local` / `--no-observe` to override a `true` default from
the file.

## Settings (`fusion.json`)

```json
{
  "version": "fusionkit.fusion.v2",
  "tool": "codex",
  "panel": [
    { "id": "gpt", "model": "gpt-5.5", "provider": "openai", "keyEnv": "OPENAI_API_KEY" },
    { "id": "sonnet", "model": "claude-sonnet-4-6", "provider": "anthropic", "keyEnv": "ANTHROPIC_API_KEY" },
    { "id": "gemini", "model": "gemini-2.5-pro", "provider": "google", "keyEnv": "GEMINI_API_KEY" }
  ],
  "judgeModel": "gpt-5.5",
  "local": false,
  "observe": false,
  "onRateLimit": "fusion",
  "budgetUsd": 5,
  "portless": true,
  "port": null
}
```

### Fields

| Field | Type | Notes |
| --- | --- | --- |
| `version` | string | Must be `"fusionkit.fusion.v2"`. A `"fusionkit.fusion.v1"` file is still accepted and upgraded. |
| `tool` | `codex` \| `claude` \| `cursor` \| `serve` | Default agent when none is passed. |
| `panel` | array | Panel models (see below). |
| `judgeModel` | string | Model used for judge synthesis. Defaults to the first panel model. |
| `local` | boolean | Use the local MLX trio by default. |
| `observe` | boolean | Boot the observability dashboard by default. |
| `onRateLimit` | `fusion` \| `passthrough` \| `fail` | What to do when a passthrough vendor hits a rate limit, quota error, or billing error. Defaults to `fusion`. |
| `budgetUsd` | number | Optional session spend cap in gateway-observed USD. |
| `portless` | boolean | Use stable portless URLs when available. Defaults to `true`. |
| `port` | number \| null | Fixed gateway port (otherwise ephemeral). |

### Panel entries

| Field | Type | Notes |
| --- | --- | --- |
| `id` | string | Stable identifier for the panel member. |
| `model` | string | The model name. |
| `provider` | `mlx` \| `openai` \| `anthropic` \| `google` \| `openai-compatible` | Defaults to `mlx`. |
| `keyEnv` | string | Env var holding this model's API key (cloud only). |
| `baseUrl` | string | Optional custom base URL. |

When `panel` is omitted, the default cloud panel is the three-vendor trio shown
above. Set `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, and `GEMINI_API_KEY` for the
strongest default behavior. If one member is missing a key or fails during a run,
the surviving members can still be fused.

## Inspect and export the effective config

Use `config show` when a run does not behave the way you expect. It prints the
merged config and where each value came from.

```bash
fusionkit config show
fusionkit config path
```

The Node CLI derives the Python router YAML from `.fusionkit/fusion.json`.
Generate that YAML only when you want to run the raw Python server directly.

```bash
fusionkit config export-yaml
fusionkit config export-yaml -o router.yaml
fusionkit serve --config router.yaml
```

## Prompt overrides (`prompts/`)

Each `.fusionkit/prompts/<id>.md` file overrides one built-in system prompt used
by the synthesizer. A file that exists and is **non-empty** replaces the
default; an empty or missing file falls back to the built-in.

| File | Overrides |
| --- | --- |
| `judge.md` | The judge analysis prompt (compares candidate trajectories). |
| `synthesizer.md` | The synthesizer prompt that produces the fused output (a final answer, or the next agentic step when tools are present). |

To regenerate the built-in defaults at any time (e.g. after upgrading), dump
them from the synthesizer package:

```bash
uvx fusionkit prompts dump --dir .fusionkit/prompts
```

<Callout type="warn">
  Never put API key values in `.fusionkit/`. Store only the env-var name in
  `keyEnv` and export the secret (or put it in `.env`).
</Callout>
