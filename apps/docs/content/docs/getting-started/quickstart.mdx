## 1. Install, provision, and check your environment

```bash
npm install -g @fusionkit/cli
fusionkit setup
fusionkit doctor
```

Fix anything marked as missing. For the default cloud panel, the provider keys
are `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, and `GEMINI_API_KEY`. `fusionkit setup`
warms the Python engine before the first session.

## 2. Launch a coding agent

From inside your git repo:

```bash
cd your-git-repo
fusionkit codex      # or: fusionkit claude | fusionkit cursor
```

On a TTY you will see a live boot checklist as the panel, synthesizer, and
gateway come up. The cloud panel asks for a one-line cost confirmation unless you
pass `--yes`. Then your agent launches pre-wired to the gateway. One `Ctrl+C`
tears the stack down.

A healthy launch prints three useful things before handing control to the agent:

- the gateway URL or portless name the agent will use;
- the fused model id, normally `fusion-panel`;
- a temp log directory for per-process output if anything later fails.

If the first run pauses while provisioning the Python engine, let it finish once
or run `fusionkit doctor --provision` ahead of time on new machines.

## 3. (Optional) Save per-repo defaults

So your team can just run `fusionkit codex`:

```bash
fusionkit init
```

This writes a committed `.fusionkit/` folder (settings plus editable prompts).
See [Configuration](/docs/getting-started/configuration).

## Common flags

- `--local`: use the on-device MLX panel instead of the cloud panel.
- `--model ID=PROVIDER:MODEL`: define a custom panel. Repeat it for multiple members.
- `--judge-model <model>`: pick the synthesis judge.
- `--budget <usd>`: stop after gateway-observed spend crosses the cap.
- `--on-rate-limit fusion|passthrough|fail`: choose failover behavior for vendor throttling.
- `--continue` or `--resume <id>`: resume a durable session.
- `--observe`: boot the local trace dashboard.
- `--yes`: skip the cloud-panel cost confirmation.

fusionkit's own flags must precede the tool name; everything after the tool is
forwarded to it. See the full [command reference](/docs/cli/commands).

For non-interactive environments, combine `--yes` with a fixed port or disabled
portless registration:

```bash
fusionkit --yes --no-portless --port 8787 serve
```

## Pick fused or direct models

The gateway exposes `fusion-panel` as the fused model. It also exposes panel
member ids such as `gpt`, `sonnet`, and `gemini` as passthrough models. Choose
`fusion-panel` when you want synthesis. Choose a member id when you want to
compare the ensemble against one provider or use
[rate-limit handoff](/docs/getting-started/rate-limit-handoff) as a fallback.

Run `fusionkit models list` to inspect local MLX models, and use
`fusionkit config show` to see the effective cloud panel that will be launched
for the current repo.
