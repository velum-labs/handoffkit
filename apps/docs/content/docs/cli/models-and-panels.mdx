---
title: Models and panels
description: Choose cloud, open-weight, OpenAI-compatible, and local MLX models for a fused panel.
---

A FusionKit panel is a set of model members that answer the same task independently. The judge then compares their trajectories and synthesizes the final response. Panel quality depends on diversity, capability, and availability. A three-model panel from three vendors usually gives more useful disagreement than three variants from the same family.

## Providers

Each panel member has an `id`, `provider`, `model`, and optional credential fields.

| Provider | Use it for | Default key env |
| --- | --- | --- |
| `openai` | OpenAI models | `OPENAI_API_KEY` |
| `anthropic` | Anthropic models | `ANTHROPIC_API_KEY` |
| `google` | Gemini models | `GEMINI_API_KEY` |
| `openrouter` | Any model on [OpenRouter](https://openrouter.ai/models) with one key | `OPENROUTER_API_KEY` |
| `openai-compatible` | vLLM, LM Studio, Together, Fireworks, DeepInfra, self-hosted endpoints | Set with `--key-env` or config |
| `mlx` | Local Apple Silicon models | None |

Add members on the command line with repeated `--model` flags.

```bash
fusionkit codex \
  --model gpt=openai:gpt-5.5 \
  --model sonnet=anthropic:claude-sonnet-4-6 \
  --model gemini=google:gemini-2.5-pro
```

Use `--model-endpoint` when a member is already running behind an OpenAI-compatible HTTP server.

```bash
fusionkit codex \
  --model gpt=openai:gpt-5.5 \
  --model-endpoint llama=http://127.0.0.1:8000
```

Use `--key-env` when a member's credential does not use the provider default.

```bash
fusionkit codex \
  --model deepinfra=openai-compatible:meta-llama/Llama-3.3-70B \
  --key-env deepinfra=DEEPINFRA_API_KEY
```

## OpenRouter

The `openrouter` provider reaches any model on [OpenRouter](https://openrouter.ai/models) with a single `OPENROUTER_API_KEY` — a convenient way to build a decorrelated panel across vendors you have no direct account with. The model name is OpenRouter's `vendor/model` id, optionally with a `:variant` suffix (for example `:free`).

```bash
export OPENROUTER_API_KEY=...
fusionkit codex \
  --model sonnet=openrouter:anthropic/claude-sonnet-4.5 \
  --model kimi=openrouter:moonshotai/kimi-k2 \
  --model deepseek=openrouter:deepseek/deepseek-chat
```

## Default cloud panel

With no custom panel and no `--local`, FusionKit uses a decorrelated cloud trio.

| id | provider | model |
| --- | --- | --- |
| `gpt` | `openai` | `gpt-5.5` |
| `sonnet` | `anthropic` | `claude-sonnet-4-6` |
| `gemini` | `google` | `gemini-2.5-pro` |

Set all three keys for the strongest default behavior.

```bash
export OPENAI_API_KEY=...
export ANTHROPIC_API_KEY=...
export GEMINI_API_KEY=...
```

If a member is missing a key or fails during a run, the remaining members can still be fused. That resilience is useful, but it is not a substitute for configuring the full panel when quality matters.

## Local MLX panel

On Apple Silicon, `--local` uses a local MLX trio with no provider keys.

```bash
fusionkit codex --local
```

Manage the local cache with `fusionkit models`.

```bash
fusionkit models list
fusionkit models download mlx-community/Qwen3-1.7B-4bit
fusionkit models rm mlx-community/Qwen3-1.7B-4bit
```

`fusionkit models list` reports downloaded state, model size, and conservative RAM fit. On non-Apple-Silicon machines, local MLX is unavailable and the CLI points you back to the cloud path.

## Mixed panels

Mixed panels are useful when you want one local perspective and multiple cloud perspectives.

```bash
fusionkit codex \
  --model gpt=openai:gpt-5.5 \
  --model sonnet=anthropic:claude-sonnet-4-6 \
  --model qwen=mlx:mlx-community/Qwen3-1.7B-4bit
```

The local member still requires Apple Silicon. The cloud members still require their provider keys.

## Choosing the judge

The judge defaults to the first panel member. Choose a different judge when the first member is cheap, local, or intentionally less capable than the synthesis model you want.

```bash
fusionkit codex \
  --model fast=openai:gpt-5.5-mini \
  --model sonnet=anthropic:claude-sonnet-4-6 \
  --judge-model claude-sonnet-4-6
```

## Cost and budget

FusionKit meters gateway-observed token usage and USD estimates when pricing metadata is available. Inspect session totals with `fusionkit sessions` and cap spend with `--budget`.

```bash
fusionkit codex --budget 5
fusionkit sessions
```

Persist a budget in `.fusionkit/fusion.json`.

```json
{
  "version": "fusionkit.fusion.v2",
  "budgetUsd": 5
}
```

## Verify your panel

Use these commands before a long session.

```bash
fusionkit doctor
fusionkit status
fusionkit config show
```

`doctor` checks prerequisites and keys. `status` previews the run plan. `config show` explains which values came from flags, `.fusionkit/fusion.json`, or built-in defaults.
