FusionKit documentation is organized by reader intent. User-facing product docs live in this site; maintainer, benchmark, protocol, release, and legacy archive docs live in the repository Markdown tree.

## The taxonomy

| Category | Reader question | Page behavior |
| --- | --- | --- |
| Orientation | Where am I, and what should I read next? | Explain scope and route readers to the right section. |
| Task guides | How do I complete this workflow? | Provide commands, expected output, and the next troubleshooting step. |
| Concepts | How does the product fit together? | Explain mental models, boundaries, and architecture. |
| Reference | What exactly does this surface contain? | Document commands, fields, packages, routes, examples, and stable interfaces. |
| Privacy | What is stored, and where does code go? | Explain local retention, provider egress, failover expansion, and telemetry behavior. |
| APIs | How do clients call FusionKit? | Describe runtime routes and generated contracts. |

## Entry justification

A page belongs in the public site when it helps a user install FusionKit, run a session, configure a repo, understand product behavior, integrate an endpoint, control cost, reason about privacy, run a product example, or call an API. Implementation ownership, generated code internals, release process, benchmark maintenance, and legacy governance material belong in repository maintainer docs.
