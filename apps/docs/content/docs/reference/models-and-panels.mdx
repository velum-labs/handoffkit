---
title: Models and ensembles
description: RouteKit endpoints and FusionKit v4 ensembles.
---

RouteKit owns models; FusionKit owns ensembles.

Define provider-facing details once:

```yaml
# .routekit/router.yaml
endpoints:
  - endpointId: fast
    model: provider-fast-model
    provider: openai-compatible
    baseUrl: https://provider.example/v1
    dialect: openai
    apiKeyEnv: PROVIDER_API_KEY
  - endpointId: deep
    model: provider-deep-model
    provider: openai-compatible
    baseUrl: https://provider.example/v1
    dialect: openai
    apiKeyEnv: PROVIDER_API_KEY
```

For a subscription login, use an account-backed endpoint instead:

```yaml
# .routekit/router.yaml
endpoints:
  - endpointId: opaque-subscription-peer
    model: gpt-5.5-codex
    account: codex
accounts:
  codex:
    enabled: true
```

The only subscription kinds are `claude-code` and `codex`. URL-backed
endpoints keep `baseUrl`, `dialect`, and optional `apiKeyEnv`; account-backed
endpoints do not accept those fields.

Compose only opaque IDs in Fusion:

```json
{
  "version": "fusionkit.fusion.v4",
  "router": { "config": ".routekit/router.yaml" },
  "ensembles": {
    "default": {
      "members": ["fast", "deep"],
      "judge": "deep",
      "synthesizer": "deep"
    }
  }
}
```

The default ensemble is advertised as `fusion-panel`; other names become
`fusion-<name>`.

```bash
fusionkit ensemble add review \
  --member fast --member deep --judge deep --synthesizer deep
fusionkit config set defaultEnsemble review
```

Use `routekit <tool> <endpoint-id>` for a single-model session. FusionKit has no
direct mode or provider/model/key launch flags.

An explicit unknown endpoint ID fails instead of using the router default.

`fusionkit models` remains the lifecycle command for the Fusion-owned local MLX
cache; local endpoints still need to be exposed through RouteKit before an
ensemble can reference them.
